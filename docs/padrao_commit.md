| Tipo | Descrição | Exemplo |
| --- | --- | --- |
| **feat** | Quando da adição de um recurso, uma *feature* (funcionalidade). | `feat: Implementação do módulo de chat com WebSocket Gateway` |
| **fix** | Correção de um bug. | `fix: Correção no escalonamento de tickets entre grupos` |
| **docs** | Atualização de documentação. | `docs: Inclusão do diagrama de arquitetura do Service Desk` |
| **style** | Mudança de formatação, sem afetar o código. | `style: Padronização de DTOs no módulo de autenticação` |
| **refactor** | Refatoração do código, sem alterar funcionalidade. | `refactor: Separação da lógica de triagem em service dedicado` |
| **test** | Adiciona ou modifica testes. | `test: Criação de testes unitários para o módulo de tickets` |
| **chore** | Atualizações menores que não impactam diretamente a funcionalidade do código. | `chore: Atualização das dependências do NestJS` |
| **ci** | Alterações nos pipelines de Integração Contínua e Deploy Automático. | `ci: Configuração do workflow para rodar migrations no PostgreSQL e seeds no MongoDB` |
